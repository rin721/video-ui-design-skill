# 设计原则

## 清晰先于氛围

每个区块都必须通过层级、分组和间距表达用途。装饰可以支持情绪，但不能与内容和控件竞争。

## 柔和强调

品牌色用于 active 状态、主操作、区块标记和 focus ring。背景强调应保持低不透明度，使界面保持可读和可替换。

## 密集但透气

内容密集型布局应使用紧凑卡片、稳定图片比例和简洁 metadata。保留外部空间与行节奏，让密集网格依然平静。

## 图标优先的工具操作

重复导航和工具操作使用图标。图标需要可访问标签，必要时提供 tooltip，并具备清晰 selected 状态。

## Token 驱动实现

所有颜色、字体、间距、圆角、阴影和动效决策都使用语义化 token。primitive token 可以替换，而无需重写组件。

## 安全响应式

固定导航尺寸保持可预测。用 minmax 规则、稳定比例和足够大的移动端点击区域约束网格。

## 默认可访问

通过对比度、focus 可见性、键盘可达性、语义 landmarks、替代文本、减少动效和清晰错误恢复，达到 WCAG 2.2 AA 或更高标准。
