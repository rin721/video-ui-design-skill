# Originality Checklist

## Required Checks

- No brand names, product names, route labels, protected marks, or distinctive phrases.
- No protected images, screenshots, logos, mascots, proprietary icons, or unique artwork.
- No unique CSS selectors, component names, file names, or navigation wording from an existing product.
- No pixel-matched layout values that only fit one page.
- Colors are converted into semantic palette roles.
- Layouts are converted into flexible grid and shell rules.
- Components are converted into generic schemas with replaceable anatomy.
- Copy is converted into content roles.
- Media is converted into composition and replacement guidelines.
- Interactions are converted into behavior patterns.
- Brand expression is converted into adjustable dimensions.

## Pass Criteria

- The package can stand alone as a new design system.
- The package can produce multiple site types.
- The package contains no external brand traces.
- The package remains useful after swapping colors, content, images, and brand voice.
