# SaaS Pattern

## Structure

```yaml
saas:
  product_nav:
    purpose: "surface product, pricing, docs, customers, and account actions"
  value_hero:
    purpose: "state the job-to-be-done and measurable outcome"
  product_clarity:
    purpose: "show workflow, interface, or system diagram"
  feature_grid:
    purpose: "explain capabilities by user task"
  proof:
    purpose: "show customer outcomes, metrics, or security trust"
  pricing_or_plan:
    purpose: "connect value to buying model"
  signup_cta:
    purpose: "start trial, book demo, or contact sales"
```

## Layout

- Use balanced density and direct copy.
- Use real interface media or abstract workflow diagrams that can be replaced.
- Keep proof near signup and pricing.

## Components

Use `NavigationBar`, `HeroSection`, `FeatureCard`, `MediaCard`, `MetricCard`, `PricingBlock`, `TestimonialBlock`, `CTASection`, and `Footer`.

## Acceptance

- Value proposition is understandable in one sentence.
- Product mechanism is visible before pricing.
- Primary signup path is repeated only at major decision points.
