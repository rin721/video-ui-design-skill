# Community Pattern

## Structure

```yaml
community:
  community_hero:
    purpose: "说明共同目的和参与价值"
  value_statement:
    purpose: "解释适合谁以及能做什么"
  discussion_highlights:
    purpose: "展示活跃主题、问题或更新"
  member_proof:
    purpose: "展示参与度、信任和势能"
  join_cta:
    purpose: "邀请加入或贡献"
  footer:
    purpose: "提供规则、治理和支持链接"
```

## Layout

- 使用 balanced 到 dense 内容，活动信号可见。
- 根据时间和回复深度，discussion card 可使用网格或列表。
- 加入路径清晰但不过度施压。

## Components

使用 `HeroSection`、`DiscussionCard`、`MemberAvatar`、`ActivityFeed`、`ReactionButton`、`Tag`、`CTASection` 和 `Footer`。

## Acceptance

- 加入前即可理解社区价值。
- 活动信号易读且不只靠颜色。
- 参与控件具备可访问名称和 selected 状态。
