# SaaS Pattern

## Structure

```yaml
saas:
  product_nav:
    purpose: "展示产品、定价、文档、客户和账户操作"
  value_hero:
    purpose: "说明待完成任务和可衡量结果"
  product_clarity:
    purpose: "展示工作流、界面或系统图"
  feature_grid:
    purpose: "按用户任务解释能力"
  proof:
    purpose: "展示客户结果、指标或安全信任"
  pricing_or_plan:
    purpose: "连接价值和购买模型"
  signup_cta:
    purpose: "开始试用、预约演示或联系销售"
```

## Layout

- 使用 balanced density 和直接文案。
- 使用真实界面媒体或可替换的抽象工作流图。
- proof 靠近 signup 和 pricing。

## Components

使用 `NavigationBar`、`HeroSection`、`FeatureCard`、`MediaCard`、`MetricCard`、`PricingBlock`、`TestimonialBlock`、`CTASection` 和 `Footer`。

## Acceptance

- 一句话即可理解价值主张。
- 定价前展示产品机制。
- 主注册路径只在关键决策点重复。
