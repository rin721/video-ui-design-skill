# Icon Guidelines

## Icon System

```yaml
icon_system:
  stroke_width: "1.75px to 2px"
  corner_style: "rounded joins and caps"
  filled_or_outline: "outline by default, filled only for selected or status emphasis"
  size_scale:
    sm: "16px"
    md: "20px"
    lg: "24px"
```

## Rules

- Use icons for repeated actions, utilities, filters, and compact navigation.
- Pair unfamiliar icons with labels or tooltips.
- Keep icon-only controls at least `40px` on desktop and `44px` on mobile.
- Selected icons may use filled treatment plus text or active marker.
- Status icons must be paired with text, not color alone.
- Use one icon family per interface.
- Avoid proprietary symbols and brand marks.

## Common Categories

- Navigation: home, search, categories, saved, history, settings, account.
- Content: article, media, product, document, collection, upload.
- Feedback: info, success, warning, danger, loading.
- Commerce: cart, favorite, compare, shipping, payment.
- Documentation: code, copy action, external link, previous, next.
